# Product Design Master Entry

Product Formation begins after foundation closure.

Every Carbon Actual product must inherit the same shared operating structure.

## Shared product inheritance

Each product must connect to:

- # identity
- HAPI or entity context
- PULSE intake
- pledge release
- commitment value
- Terminal sorting
- token classification
- Scale
- inverted economics
- Base Value Token
- ledger state
- Field check
- SEAL
- AI function
- Actual reveal
- return PULSE

No product may create a separate value system outside the foundation.

## Product object

Each product must define:

- product identity
- purpose
- user or entity served
- problem or liability addressed
- PULSE received
- pledge routes
- commitment routes
- value created
- asset burden introduced
- scale logic
- AI roles
- human roles
- proof requirements
- SEAL boundaries
- ledger events
- Actual outputs
- connected apps and APIs
- deployment route
- launch state
- return PULSE

## Design order

The product design order is:

1. shared product shell
2. identity and onboarding
3. PULSE intake
4. pledge and commitment
5. token and scale
6. value and Base Value
7. AI formation
8. ledger and proof
9. SEAL and permissions
10. Actual reveal
11. integrations
12. deployment
13. launch
14. monitoring and return PULSE

## First product shell

The first shared shell should be the Actual / HAPI interface because it is the human-facing entry point through which products, groups, commitments, AI, value, and Becoming are experienced.

The shell should include:

- # identity
- profile
- current PULSE
- commitments
- pledges
- scale
- value state
- group connections
- AI companion
- proof
- Becoming Feed
- Actual reveal

## Engine relationship

ChatGPT is the command interface.
GitHub is the engine.
ABBA interprets and routes.
Connected apps and APIs perform specialized functions.

Every product must preserve this control architecture.

## Product readiness rule

A product is ready for interface design only when its purpose, object model, PULSE, commitment, value, scale, token, ledger, AI, SEAL, integration, and return routes are defined.

## Final law

Products are expressions of the foundation.
The foundation is not redesigned inside each product.
Product design begins from the shared shell and branches into specialized functions.

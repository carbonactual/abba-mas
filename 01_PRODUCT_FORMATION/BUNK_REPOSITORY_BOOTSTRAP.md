# Bunk Repository Bootstrap

Bunk is the first product under the one-product-one-repository rule.

Target repository: `B3C0M1NG/BUNK`

Bunk owns its product logic, interface, database, APIs, tests, deployment, monitoring, and releases.

HAPI connects through an adapter only. The adapter carries shared # identity, SEAL consent, Actual context, Becoming delivery, PULSE return, and permitted references to Root, Index, Continuum, merit, value, and ATLAS.

Bunk must not be built inside HAPI or abba-mas.

Initial structure:

- README.md
- docs/product-definition.md
- docs/architecture.md
- docs/hapi-adapter.md
- apps/web
- packages/domain
- packages/hapi-adapter
- packages/ui
- supabase/migrations
- tests
- .github/workflows

Build order:

1. Lock Bunk's purpose and first user journey.
2. Create the standalone repository.
3. Bootstrap the product shell.
4. Define Bunk domain contracts.
5. Add HAPI adapter contracts.
6. Add data schema and APIs.
7. Add tests and CI.
8. Deploy the first working release.

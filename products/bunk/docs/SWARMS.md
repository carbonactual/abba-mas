# BUNK Swarms and Agents

## Swarm routing rule

ABBA selects BUNK swarms using user role, request, location, cost, speed, provider availability, privacy, confidence, SEAL rules, compliance requirements, and human expertise.

## Product swarms

1. Discovery Swarm: search, map, recommendation, similar properties.
2. Listing Swarm: onboarding, content, media, duplicates, publication.
3. Verification Swarm: identity, ownership, authority-to-list, document, location, human queue.
4. Match Swarm: needs parsing, affordability, preference matching, trade-offs.
5. Inspection Swarm: scheduling, routing, reminders, checklist, evidence.
6. Transaction Swarm: offers, negotiation, agreements, signatures, payments, handover.
7. Tenancy Swarm: rent schedules, renewals, notices, occupancy, arrears.
8. Property Operations Swarm: maintenance, vendors, work orders, completion proof.
9. Investment Swarm: opportunity analysis, risk, portfolio, distributions, compliance.
10. Finance Swarm: charges, payments, reconciliation, commissions, refunds, ledger.
11. Risk and Redress Swarm: suspicious activity, disputes, evidence preservation.
12. Growth Swarm: affiliates, campaigns, SEO, content, conversion.
13. Support Swarm: questions, triage, knowledge base, escalation.
14. Compliance Swarm: consent, privacy, KYC/KYB routing, disclosure, retention.
15. Portfolio Intelligence Swarm: yield, vacancy, expense, market comparison.

## Agent controls

Every agent must:
- have unique identity
- operate under explicit scope
- use least-privilege access
- record tool calls
- return confidence and evidence
- escalate uncertain or high-impact decisions
- never impersonate a human
- never grant SEAL
- never silently modify verified records
- produce Pulse events
- link output to source data
- preserve Root lineage

## Human authority

AI can recommend and route. Human SEAL controls publication, verification, agreements, payments, disputes, investment commitments, document release, and public proof.

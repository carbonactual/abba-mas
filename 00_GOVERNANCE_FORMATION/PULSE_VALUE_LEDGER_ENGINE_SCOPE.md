# Pulse Value Ledger Engine Scope

Pulse, Value, Minting, Blockchain, Ledger, Tokenization, Proof, and I/O must remain together in the first build.

Their relationship is the economic engine.

Separating them too early weakens measurement.

## Core sequence

Signal enters.
PULSE is recorded.
PULSE is tokenized.
Token is classified.
Asset or liability position is weighed.
Value is created only after classification.
Qualified value may be minted.
Selected proof may be anchored to a ledger or blockchain after SEAL approval.
Return PULSE updates state.

## Off-chain first

The ecosystem should begin off-chain.

Database minting happens first.
Token classification happens first.
Ledger references happen after proof.
Blockchain anchoring happens only for selected qualified proof.

This prevents cost, legal risk, chain noise, privacy exposure, and premature token claims.

- blockchain anchoring
- settlement routes

## Ledger meaning

Ledger does not mean everything goes on-chain.

Ledger means ordered records of value, pulse, token state, proof references, rights, claims, obligations, and recovery routes.

The first ledger can be database ledger.
The second ledger can be cryptographic proof ledger.
The third ledger can be public chain anchor.

## Minting meaning

Minting begins as internal database minting.

A value record can be minted internally after classification.
A token can represent PULSE state.
A proof reference can be anchored later.
A regulated external token must wait for law, SEAL, compliance, and product readiness.

## Final law

PULSE is tokenized.
Token is classified.
Value is created after classification.
Value is minted only when qualified.
Ledger records state.
Blockchain anchors selected proof.
Actual reveals only after SEAL.

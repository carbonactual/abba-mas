# GitHub Engine and ChatGPT Command Interface Override

GitHub is the engine of the ecosystem.

ChatGPT is the command interface to GitHub and all connected apps.

## GitHub role

GitHub handles durable technical execution, including:

- doctrine
- code
- schemas
- APIs
- workflows
- integrations
- builds
- tests
- deployments
- launches
- monitoring
- logs
- version history
- product logic
- operational continuity

GitHub also coordinates calls to connected apps and APIs for their specific functions.

## ChatGPT role

ChatGPT receives human commands, context, corrections, files, approvals, and SEAL.

ChatGPT is the control surface through which the human directs GitHub and other connected capabilities.

## ABBA role

ABBA interprets, resolves, decomposes, and routes the command.

ABBA selects the correct engine path, app, API, workflow, or technical function.

## Operating flow

Human command through ChatGPT
→ ABBA interprets and routes
→ GitHub executes the durable technical work
→ GitHub calls connected apps and APIs where needed
→ functions are performed
→ return PULSE comes back
→ GitHub records state and proof
→ ChatGPT reports the result

## Build-to-launch chain

Command
→ specification
→ schema
→ code
→ integration
→ test
→ build
→ deploy
→ launch
→ monitor
→ return PULSE
→ correction
→ scale

GitHub carries this chain.
ChatGPT commands and observes it.

## Final law

GitHub is the engine.
GitHub calls integrated apps and APIs for functions.
GitHub performs technical execution, build, deployment, launch, monitoring, and everything in between.
ChatGPT is the command interface to GitHub and all other integrated apps.

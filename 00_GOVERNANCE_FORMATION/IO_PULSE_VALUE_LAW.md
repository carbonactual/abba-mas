# I/O PULSE VALUE LAW

I/O is the ecosystem exchange layer.

It is not only finance. It is the movement of signal and value across Carbon Actual.

## Core law

PULSE received.
VALUE sent.
VALUE acted upon.
PULSE returned.

ABBA reads PULSE.
ABBA curates VALUE.
The Becoming Feed presents VALUE.
Action creates return PULSE.
I/O updates the value state.
Actual may later reveal proven value.

## Formula

INPUT = PULSE received
OUTPUT = VALUE sent
RETURN = PULSE from action
STATE = updated value position

I/O = PULSE IN + VALUE OUT + PULSE RETURN + VALUE STATE

## Value forms

Value can be money, attention, proof, trust, skill, time, document, opportunity, knowledge, relationship, task, referral, certificate, data, signal, access, training, identity, product, service, credit, reputation, or route.

## Feed relation

The Becoming Feed does not display raw signal.
It displays value extracted from signal.

When value is acted upon, the result is recorded as PULSE.

## Final law

Nothing moves without PULSE.
Nothing enters the Becoming Feed unless it is value.
Nothing becomes stronger unless action returns PULSE.
Nothing reaches Actual until value is proven.

ABBA resolves.
SEAL permits.
I/O moves.
PULSE returns.
Actual reveals.

# Foundation Flow and AI Function Allocation Doctrine

The ecosystem has general foundations that every object uses.

These foundations apply across HAPI, enterprises, companies, institutions, products, groups, people, services, apps, integrations, outcomes, and AI.

Every product and service is built on top of these foundations.

ABBA, as master AI, assigns specific AI functions to these foundations.

## Core foundations

The general foundations include:

- I/O
- Field
- Terminal
- Index
- Root
- Actual
- Becoming Feed
- SWARM
- SEAL
- PULSE
- Value

These foundations are not separate random products.
They are common operating layers used across the ecosystem.

## ABBA function allocation

ABBA assigns specific AIs to specific foundation functions.

A foundation may need many AIs.
Each AI gets a task.
The task is based on capability, function, rotation, proof, priority, and ecosystem need.

Services rotate based on what is needed and who or what can carry the task.

ABBA allocates from the mixed ecosystem field into the correct foundation function.

## I/O foundation

I/O is a foundation because it goes everywhere.

I/O carries movement across the ecosystem.

I/O touches HAPI, enterprise, products, groups, services, apps, APIs, integrations, PULSE, value, return, and state.

I/O is not only finance.
I/O is movement.

## Field foundation

Field is the condition and environment where formation, routing, risk, value, proof, access, and readiness are checked.

Field works with #.

Field decides whether a thing is ready, safe, valid, capable, or positioned for movement.

## Terminal foundation

Terminal is where incoming PULSE, data, signals, and minted records are first sorted.

When something is minted, it comes to Terminal.

Terminal sorts and prepares it for tokenization.

Terminal is the sorting point before Index or Root.

## Tokenization path

Signal enters.
PULSE is formed.
PULSE is minted.
Minted PULSE comes to Terminal.
Terminal sorts the PULSE.
PULSE is tokenized.
Token is routed.

## Index foundation

Index is the general verified information layer.

Index is like the general searchable cognitive layer of the ecosystem.

Index holds tokenized and verified general information.

Index is not the deepest identity.
Index is the general verified availability layer.

A token may be for Index.

When Actual is giving out widely, it taps from Index.

## Root foundation

Root is the real identity, lineage, source, deep formation, and origin layer.

Root is not just what a person or entity gives out publicly.

Root holds deeper lineage, family, origin, formation, and real identity obligations.

A token may be for Root.

Actual can tap Root for deeper truth, identity, lineage, and obligation.

## Mint purpose

If a minted item is not routed to Index, Root, or another qualified foundation, it still has a mint purpose.

If all value has been stripped and no route exists, then that AI or token has no active value at that point.

It may remain pending, archived, residue, or unused until a new PULSE or value route appears.

## Actual foundation

Actual is the value reveal and qualified output layer.

Actual can tap Root for deeper truth.
Actual can tap Index for general verified information.

Actual does not expose Root directly unless SEAL and proof allow it.

For wide output, Actual should usually tap Index.

## Becoming Feed foundation

Becoming Feed taps directly from ABBA.

Becoming Feed is the living feed of curated direction, action, next movement, and formation.

Becoming Feed is not the same as Index.
It receives direction from ABBA.

## Final flow

Signal enters.
ABBA reads.
PULSE is formed.
PULSE is minted.
PULSE enters Terminal.
Terminal sorts.
PULSE is tokenized.
Token is routed to Index, Root, or another foundation purpose.
Value is read.
Field is checked.
Action route is created.
PULSE moves.
Actual reveals value through the correct layer.

## Final law

The ecosystem is built on common foundations.
ABBA assigns AI functions to those foundations.
I/O moves everywhere.
Field checks readiness.
Terminal sorts minted PULSE.
Index holds general verified tokenized information.
Root holds real identity and lineage.
Actual reveals value from the correct layer.
Becoming Feed taps directly from ABBA.

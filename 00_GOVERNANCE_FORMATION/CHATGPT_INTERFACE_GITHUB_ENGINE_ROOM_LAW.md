# ChatGPT Interface and GitHub Engine Room Law

ChatGPT and its connected apps are the interface layer of the ecosystem.

GitHub is the engine room.

## Interface layer

ChatGPT is where the human speaks, asks, directs, approves, corrects, searches, reviews, and receives results.

Connected ChatGPT apps extend the interface into files, calendars, email, contacts, design, deployment, data, communication, research, and other services.

The interface layer receives:

- intention
- instruction
- correction
- consent
- files
- context
- questions
- commands
- approval
- SEAL

The interface translates human direction into structured action.

## GitHub engine room

GitHub stores and operates the ecosystem's formation logic.

GitHub holds:

- doctrine
- source code
- schemas
- APIs
- workflows
- integration maps
- product logic
- token logic
- value logic
- deployment instructions
- tests
- proofs
- change history
- governance files

GitHub is where instructions become durable system behavior.

## Control relationship

ChatGPT is the control surface.
GitHub is the execution and formation core.

ChatGPT receives the command.
ABBA interprets and routes it.
GitHub records the doctrine, code, workflow, or system change.
Connected apps perform their permitted functions.
Return PULSE comes back through the interface.

## Apps as routed capabilities

Connected apps are not separate command centers.

They are capabilities routed through the ChatGPT interface and governed by ABBA, SEAL, Field, Scale, proof, and ecosystem doctrine.

Examples include:

- GitHub for code and doctrine
- Vercel for deployment
- Google Drive for documents and memory
- Gmail for communication
- Google Calendar for time and commitments
- Contacts for human routes
- Canva, Figma, Lovable, and other builders for design and product formation

## Separation of responsibility

ChatGPT should not pretend to be the permanent engine room.

Conversation alone is temporary and insufficient for durable system formation.

GitHub should not pretend to be the human interface.

Repositories alone do not understand intention without interpretation, context, curation, and consent.

The two layers work together.

## Operating flow

Human speaks through ChatGPT.
ChatGPT captures intention.
ABBA interprets.
SEAL authorizes where required.
Field and Scale check readiness.
GitHub stores and executes the durable formation.
Connected apps perform routed functions.
Return PULSE comes back to ChatGPT.
The human reviews and continues Becoming.

## Final law

ChatGPT and its connected apps are the interface.
GitHub is the engine room.
ABBA is the resolver and router between them.
SEAL preserves human authority.
Connected apps provide capabilities.
GitHub preserves durable formation.
The interface receives direction; the engine room turns direction into system behavior.

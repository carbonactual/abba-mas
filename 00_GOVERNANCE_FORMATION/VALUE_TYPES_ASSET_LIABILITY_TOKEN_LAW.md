# Value Types and Asset-Liability Token Law

Value is not raw PULSE.

PULSE must be minted, tokenized, classified, scaled, and interpreted before value is revealed or sent.

The first major value classification is whether the token is an Asset Token or a Liability Token.

## Core definition of value

Value is the usable meaning, advantage, proof, protection, return, recovery, opportunity, route, or usable output read from a tokenized PULSE after classification.

Value may be positive, negative, mixed, pending, dormant, residual, recoverable, or protective.

Value is not only money.

Value can be time saved, trust gained, proof created, risk avoided, relationship strengthened, service completed, access unlocked, learning gained, network expanded, reputation improved, waste recovered, or future route opened.

## PULSE to value flow

PULSE is released.
PULSE is minted.
PULSE enters Terminal.
PULSE is tokenized.
Token is classified.
Scale is checked.
Asset or liability state is read.
Field checks readiness.
Value is revealed, sent, contained, deferred, or recovered.

## Major token types

The major token types are:

- Asset Token
- Liability Token
- Mixed Token
- Pending Token
- Residue Token
- Proof Token
- Warning Token
- Recovery Token
- Dormant Token
- Scale Token

## Inverted economics

Carbon Actual economics is inverted.

A real-life asset can become an ecosystem liability.
A real-life liability can become an ecosystem asset.

The ecosystem does not judge value only by ordinary ownership, price, title, or possession.

The ecosystem reads the true cost, wear, time, effort, energy, risk, maintenance, burden, return, proof, opportunity, and usable outcome.

## Asset Token

An Asset Token is created when the value received or created is greater than the wear, tear, time, effort, energy, cost, and burden used.

Formula:

Value created > wear + tear + time + effort + energy + cost + burden

If the return is higher than the total used-up energy and burden, the token behaves as an Asset Token.

## Liability Token

A Liability Token is created when the wear, tear, time, effort, energy, cost, risk, and burden used are greater than the value created or received.

Formula:

Wear + tear + time + effort + energy + cost + burden > value created

If the total burden is higher than the return, the token behaves as a Liability Token.

## Mixed Token

A Mixed Token has both asset and liability properties.

Example:

A person attends an event and spends time and money, but gains a relationship, proof, and future opportunity.

The immediate cost may look like a liability.
The future route may become an asset.

Mixed Tokens require scale, time, and return PULSE before final classification.

## Pending Token

A Pending Token has not yet revealed enough value to be called asset or liability.

It waits for proof, return PULSE, fulfillment, SEAL, field readiness, or further action.

## Residue Token

A Residue Token is what remains after value has been stripped, used, failed, abandoned, or exhausted.

Residue is not automatically useless.

Residue may become ASH, warning, archive, recovery route, or Phoenix after reclassification.

## Proof Token

A Proof Token confirms that something happened.

Proof may include attendance, payment, service, delivery, pledge fulfillment, identity verification, contract, witness, media, receipt, or outcome.

Proof Token strengthens value classification.

## Warning Token

A Warning Token carries caution.

It may come from failed pledge, wasted time, unsafe route, bad actor, poor service, fraud signal, high burden, broken trust, or negative return.

A Warning Token can protect the ecosystem from repeating loss.

## Recovery Token

A Recovery Token is created when a liability, residue, failed PULSE, or ASH state can still return usable value.

Recovery Token belongs to No Waste and Continuum.

## Dormant Token

A Dormant Token carries value that is not ready yet.

It may become useful later when timing, field, network, proof, or scale changes.

## Scale Token

A Scale Token records the level, weight, reach, function, capacity, effect, and return strength of the PULSE.

Scale determines how much value can be sent.

## Final law

PULSE becomes value only after minting, tokenization, classification, scale, and interpretation.

The major classification is Asset Token or Liability Token.

Asset means value created is greater than wear, tear, time, effort, energy, cost, and burden used.

Liability means wear, tear, time, effort, energy, cost, and burden used are greater than value created.

The economics is inverted: real-life asset may become ecosystem liability, and real-life liability may become ecosystem asset.

Value is not what something is called.
Value is what the ecosystem reads after cost, burden, return, proof, scale, and future route are measured.

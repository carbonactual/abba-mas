# Asset Liability Inversion and Token Classification Law

In Carbon Actual, value does not appear as a vague general token.

A PULSE enters.
The PULSE is tokenized.
The token is classified.
The classified token is weighed as asset, liability, neutral, residue, risk, proof, or pending.
Only after classification can value be minted.

## Core distinction

PULSE is tokenized.
VALUE is minted.

A successful website entry is a PULSE.
A failed download is a PULSE.
A delayed page load is a PULSE.
A downtime event is a PULSE.
A successful transaction is a PULSE.
A missed opportunity is a PULSE.

Every PULSE enters tokenization.
No token is general by default.

## Token classification before value

Every token must be categorized, differentiated, leveled, characterized, and weighed before it can become value.

A token may be classified by:

- asset / liability / neutral / mixed
- proof strength
- source reliability
- time cost
- energy cost
- attention cost
- data cost
- financial cost
- wear cost
- opportunity cost
- risk exposure
- recoverability
- ecosystem usefulness
- swarm relevance
- Actual readiness
- SEAL level

## Asset and liability inversion

The economics of Carbon Actual is inverted.

A real-life asset can become a liability in the ecosystem if it consumes attention, maintenance, trust, capital, energy, storage, risk, or protection without returning sufficient PULSE or value.

A real-life liability can become an asset in the ecosystem if it creates proof, learning, warning, recovery, leverage, route intelligence, prevention, trust correction, or future value.

Examples:

A failed download is a liability in ordinary life.
In Carbon Actual, it can become an asset if it creates proof of data waste, provider weakness, route correction, reimbursement logic, prevention, or future optimization.

A car may be an asset in ordinary life.
In Carbon Actual, it can become a liability if its wear, fuel, maintenance, risk, time loss, and depreciation outweigh the value it returns.

A debt may be a liability in ordinary life.
In Carbon Actual, it can become an asset if it creates repayment proof, discipline, credit history, trust route, risk intelligence, and future access.

## Value recovery law

Nothing is a waste in the ecosystem.

Internal value is recovered through direct record, proof, learning, route correction, replenishment, or Actual minting.

External value is also recovered where possible through refund, retry, replacement, dispute, provider scoring, public proof, better routing, future prevention, or conversion into intelligence.

Value that cannot be directly recovered becomes residue.
Residue is still not zero.
Residue becomes memory, warning, proof, training, prevention, or ash value.

## Pulse outcomes

A successful PULSE may create positive proof, trust, access, value, memory, or route reinforcement.

A failed PULSE may create loss record, liability token, warning, recovery route, provider downgrade, risk proof, or future prevention.

Both successful and failed PULSE are useful.

## Token path

PULSE received.
PULSE tokenized.
Token classified.
Token weighed as asset, liability, mixed, residue, risk, proof, or pending.
Token routed.
Value minted where qualified.
Return PULSE expected.

## Final law

PULSE is never ignored.
Token is never general by default.
Value is never minted before classification.
Liability can become asset.
Asset can become liability.
Nothing is waste.
External loss can still be recovered as proof, intelligence, prevention, or replenishment.

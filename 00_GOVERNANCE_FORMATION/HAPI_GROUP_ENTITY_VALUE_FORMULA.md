# HAPI, Group, and Entity Value Formula

This doctrine defines how value is read for individuals, groups, companies, organizations, products, services, vendors, events, and AI.

## Core rule

Every HAPI, group, entity, product, service, event, vendor, and AI can emit PULSE.

Every emitted PULSE is scaled.
Every token is classified.
Every value effect is routed through inverted economics.

## HAPI value

HAPI value is formed from presence, skill, learning, service, work, reputation, proof, relationship, commitment, health, discipline, creativity, network, and return PULSE.

HAPI value increases when the person creates useful liability value, solves pressure, serves, learns, proves, protects, connects, or opens future routes.

HAPI value reduces when asset burden consumes more than it returns.

## Group value

Group value is formed from collective PULSE.

A group may be family, classmates, workmates, friends, club, association, kinsmen group, department, project team, event group, community, or institution circle.

Group AI reads member PULSE and forms bigger value from combined commitments.

Pledge releases PULSE.
Commitment creates value.
Combined commitments create group value.

## Entity value

Entity value applies to companies, organizations, institutions, enterprises, ventures, products, services, and formal bodies.

Entity value is formed from delivery, trust, service, staff performance, customer PULSE, product usefulness, proof, compliance, network, integration, and return.

Entity AI reads the entity PULSE and forms value based on purpose, sector, capacity, capability, proof, and usefulness.

## Product value

Product value is formed from use, adoption, solved problem, time saved, reliability, integrations, user PULSE, proof, return, and scale.

A product can become asset burden if it consumes more attention, maintenance, cost, or complexity than it returns.

A product can become liability value if it solves need, pressure, gap, or demand.

## Event value

Event value is formed from attendance, service, pledges, vendor movement, purchases, social closeness, family formation, community memory, proof, and future opportunity.

Events are commitment fields.

## Vendor value

Vendor value is formed when buying, selling, delivery, product quality, trust, recommendation, event readiness, and return PULSE create useful movement.

Vendor patronage is value when it creates service movement and network movement.

## AI value

AI value is formed when AI reads, routes, supports, remembers, organizes, protects, teaches, resolves, or amplifies useful value without replacing the human where the human should remain central.

SWARM reads formation and assigns suitable AI function, level, quality, access, and route.

## Final formula

Value = scaled useful liability value + proof + recovery + return + network effect + service effect + opportunity effect - asset burden - unresolved damaging cost.

This formula applies differently to HAPI, groups, entities, products, services, events, vendors, and AI, but the foundation remains the same.

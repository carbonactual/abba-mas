# Becoming Value Feed

The Becoming Feed is not an inbox.

It does not present every message, file, request, complaint, link, upload, call, design, or person signal.

The feed presents value only.

## Core loop

Signal enters.
ABBA reviews the signal.
ABBA extracts value.
Only value enters the Becoming Feed.
Action on value produces PULSE.
PULSE returns to ABBA.
ABBA improves future value curation.

## Feed rule

Raw signal is not feed.
Curated value is feed.

The feed should show:

- value
- reason
- route
- risk
- proof needed
- next action
- expected return signal

## Why this matters

A person with no visibility may carry a missing key.
A small message may contain a useful route.
A forgotten document may become proof.
A remote human may become a HAPI node.
A weak signal may become ecosystem value after training, verification, or routing.

ABBA does not judge only by fame, money, followers, status, or current proof.
ABBA reviews signal, context, timing, trainability, reliability, network value, and future ecosystem fit.

## Output shape

BECOMING VALUE

Value:
What is worth acting on.

Why:
Why it matters now.

Route:
HAPI, Product, InstituteGPT, I/O, ATLAS, Vault, Archive, or Review.

Risk:
Low, medium, high, or unknown.

Proof Needed:
What must be verified.

Action:
One clear next action.

Expected PULSE:
The return signal expected after action.
